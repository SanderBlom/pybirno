# Security policy

## Reporting a vulnerability

If you discover a security vulnerability in this library, please report it responsibly.

### BIR API issues

For security issues related to the BIR API itself (for example, data exposure or authentication flaws in the upstream service), please contact BIR directly at [bir@bir.no](mailto:bir@bir.no).

### Library issues

For security issues in this library that should not be disclosed publicly (for example, unpatched exploits or vulnerabilities that could be abused before a fix is available), please contact the maintainer privately at [support@blomvag.no](mailto:support@blomvag.no).

Please do **not** open a public GitHub issue for security vulnerabilities. This gives us time to investigate and release a fix before the issue becomes widely known.

### What to expect

- You will receive an acknowledgment within 48 hours.
- We will work with you to understand the issue and determine a fix.
- A security advisory will be published once a fix is available.
